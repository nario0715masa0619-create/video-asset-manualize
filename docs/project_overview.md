# Project Overview

## 1. プロジェクト概要

**VideoAsset Manualize** は、既存の研修動画・マニュアル動画を、再利用可能な教育資産・業務資産へ変換するためのプロジェクトです。

本プロジェクトでは、動画を単なる視聴コンテンツとして扱うのではなく、**構造化 JSON・要約・HTML / PDF マニュアル・FAQ 候補・チェックリスト候補**へ展開可能な資産として再整理することを目指します。

中心となる考え方は、次の変換です。

**動画 → 証跡付きテキスト化 → 構造化資産化 → マニュアル / PDF / 再利用データ化**

---

## 2. 背景

多くの企業では、すでに研修動画や業務手順動画が存在しています。  
一方で、それらは次のような課題を抱えたまま運用されがちです。

- 動画はあるが、必要な情報をすぐに探せない
- 新人教育が「まず動画を見て」で終わりやすい
- 動画内のノウハウがテキスト資産として整理されていない
- 文字起こしだけでは、現場で使える手順書にならない
- OJT や引き継ぎが属人化しやすい
- 注意点や NG 例が動画の中に埋もれてしまう

VideoAsset Manualize は、こうした状態を解消し、動画を**「見る教材」から「使える教材」へ変換する**ことを狙います。

---

## 3. 解決したい課題

本プロジェクトが主に解決対象とするのは、以下の課題です。

### 3.1 動画の検索性が低い

動画は連続視聴が前提になりやすく、必要な箇所だけを素早く確認しにくいという問題があります。

### 3.2 教育品質がばらつく

担当者ごとの説明に依存すると、手順の粒度や強調する注意点に差が出やすくなります。

### 3.3 ノウハウが再利用しにくい

動画に閉じた状態では、FAQ、チェックリスト、社内検索、QA ボットなどへの二次利用が難しくなります。

### 3.4 文字起こしだけでは不十分

全文テキストがあっても、実務で必要なのは「何を、どの順番で、何に注意して行うか」が分かる構造化情報です。

---

## 4. プロジェクトの目的

VideoAsset Manualize の目的は、既存動画から実務利用可能な成果物を安定して生成できる基盤を作ることです。

具体的には、以下を目指します。

- 動画から業務手順を抽出し、再現可能な粒度で構造化する
- 説明部分を要約し、背景や目的を短く把握できるようにする
- 注意点、NG 例、よくあるミスを明文化する
- `training_asset_spec` を正本として各種成果物を派生生成できるようにする
- HTML / PDF マニュアルへ展開できる状態を作る
- 将来的な FAQ、チェックリスト、検索基盤への再利用を可能にする

---

## 5. このプロジェクトの立ち位置

VideoAsset Manualize は、次のようなプロジェクトではありません。

- 新しい動画を制作するプロジェクト
- 文字起こしだけを提供するプロジェクト
- LMS / eラーニングシステム本体を構築するプロジェクト

代わりに、次の立ち位置を取ります。

- 既存動画を整理・構造化するプロジェクト
- 手順・注意点・証跡を保持した教育資産化基盤
- マニュアル、FAQ、チェックリスト等への展開を支える中間データ基盤

---

本プロジェクトは、主に以下の場面での活用を想定しています。

## 8. 提供価値

### 8.1 動画を使いやすい形に変える

動画のままでは見返しに時間がかかりますが、要約・手順・注意点が整理されていれば、必要な情報だけを素早く参照できます。

### 8.2 教育コストを下げる

- 同じ説明の繰り返しを減らせる
- 動画の全編再視聴を減らせる
- 動画と手順書を併用しやすくなる

### 8.3 業務品質を標準化する

- 手順の粒度を揃えやすくなる
- 注意点や NG 例を明文化できる
- 拠点や担当者ごとの差異を把握しやすくなる

### 8.4 ナレッジの二次利用を可能にする

構造化データとして保持することで、将来的に以下へ展開しやすくなります。

- FAQ
- チェックリスト
- 社内検索
- QA ボット
- ダイジェスト版ハンドブック

---

## 9. 基本方針

### 9.1 正本は `training_asset_spec`

本プロジェクトでは、動画から直接 PDF を作るのではなく、まず **`training_asset_spec`** に正規化し、そこから成果物を派生生成します。

### 9.2 要約と手順抽出を分離する

説明や背景は要約対象ですが、実操作・条件分岐・注意点は要約しすぎず、構造化して保持します。

### 9.3 証跡を保持する

最終成果物だけでなく、元動画のどこから抽出したかを追えることを重視します。  
そのため、以下のような証跡を扱います。

- 発話開始 / 終了時刻
- transcript
- OCR
- スクリーンショット候補
- 話者情報
- 抽出信頼度

### 9.4 出力は派生生成する

HTML / PDF / FAQ / チェックリストは、正本 JSON から派生する成果物として扱います。

## 11. 成功条件

本プロジェクトでは、少なくとも以下を満たす状態を成功とみなします。

- 主要な手順が抜け落ちていない
- 新人でも再現可能な粒度で手順化されている
- 注意点や事故防止観点が保持されている
- 元動画との対応関係を追える
- マニュアルとして読める HTML / PDF が生成できる
- FAQ / チェックリスト等へ再利用できる構造になっている

---

## 12. 非ゴール

初期段階では、以下は対象外とします。

- 動画編集や新規動画制作
- LMS / eラーニング基盤の構築
- 高度なブランドデザインテンプレートの量産
- 完全自動化のみで人手レビューを不要にすること
- 初期段階からの大規模 SaaS 化
- あらゆる種類の動画への一括対応

---

## 14. まとめ

VideoAsset Manualize は、動画を単なる視聴素材として消費するのではなく、  
**組織で再利用できる教育資産・業務資産へ変換するための基盤**です。

目指すのは、  
**「動画はあるが活かしきれていない状態」** を、  
**「検索できる・読める・配れる・引き継げる状態」** へ変えることです。
