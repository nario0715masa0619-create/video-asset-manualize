# MVP Scope

## 1. 目的

このドキュメントは、**VideoAsset Manualize** の MVP（Minimum Viable Product）における対象範囲を定義するものです。

本プロジェクトでは、既存の研修動画・マニュアル動画を、**構造化 JSON・要約・HTML / PDF マニュアル**へ変換可能な状態にすることを最初の目標とします。

MVP では、最初から多機能化を目指すのではなく、**1本の動画を実務利用可能な最小成果物へ安定して変換できること**を重視します。

---

## 2. MVP の定義

VideoAsset Manualize における MVP は、次の最小単位で定義します。

**1動画 → 1構造化JSON → 1要約 → 1HTML / PDF マニュアル**

ここでいう「成立」とは、単に出力ファイルが作られることではなく、以下を満たすことを指します。

- 動画の主要手順が構造化されている
- 説明部分と操作手順が分離されている
- 注意点や事故防止観点が一定程度保持されている
- 元動画との対応を最低限追跡できる
- 配布・レビュー可能な HTML / PDF が生成できる

---

## 3. MVP の狙い

MVP で検証したいのは、主に以下の 4 点です。

### 3.1 手順化の成立性

動画の内容を、章・手順・ステップの形で再構成できるかを確認します。

### 3.2 要約と手順抽出の分離

説明や背景は要約しつつ、実操作や条件分岐は落とさず保持できるかを確認します。

### 3.3 証跡との接続

抽出した内容が transcript や時刻情報などの証跡と最低限結びついているかを確認します。

### 3.4 出力物としての実用性

生成された HTML / PDF が、社内レビューや簡易マニュアルとして読める状態かを確認します。

---

## 4. MVP の対象範囲

MVP で対象とするのは、**単一動画を入力として、最小限の教育資産を生成する範囲**です。

### 4.1 対象に含めるもの

- 単一動画の入力
- 動画の文字起こし
- 必要最低限の OCR 抽出
- セグメント分類
- 要約抽出
- 手順抽出
- 注意点抽出
- `training_asset_spec` 生成
- HTML マニュアル生成
- PDF マニュアル生成

### 4.2 必須ではないが、入ってもよいもの

- チェックリスト候補の簡易抽出
- FAQ 候補の簡易抽出
- スクリーンショット候補の保持
- レビュー補助情報の追加

---

## 5. MVP の入力条件

MVP では、すべての種類の動画を対象にするのではなく、比較的扱いやすい動画から始めます。

### 5.1 推奨する入力動画

以下の条件を満たす動画を、MVP の対象として推奨します。

- テーマが比較的単一である
- 業務手順や操作手順が明確である
- 音声が比較的聞き取りやすい
- 画面上の情報がある程度読み取れる
- 手順の開始と終了が把握しやすい
- 10〜30分程度で扱いやすい長さである

### 5.2 MVP 初期では避けたい動画

以下のような動画は、初期 MVP では優先度を下げます。

- 雑談や対談が中心の動画
- 手順より概念説明が中心の動画
- 音声品質が低い動画
- 画面変化が少なく OCR 効果が薄い動画
- 1本の中に複数テーマが混在している長時間動画
- 現場ノイズや複数話者が多く、切り分けが難しい動画

---

## 6. MVP の出力物

MVP の必須出力物は、以下の 4 点です。

### 6.1 構造化 JSON

`training_asset_spec` を正本として生成します。

最低限、以下を含むことを想定します。

- `asset_meta`
- `source_evidence`
- `instructional_core`
- `_metadata`

### 6.2 要約テキスト

動画全体の背景・目的・概要を短く把握できる要約です。  
ここでは、説明系セグメントを中心に扱います。

### 6.3 HTML マニュアル

レビューしやすく、ブラウザで確認しやすい形式のマニュアルです。

### 6.4 PDF マニュアル

配布・保存しやすい形式のマニュアルです。  
MVP ではシンプルなレイアウトで十分とします。

### 6.5 付随出力

必要に応じて、以下を保持しても構いません。

- 生の文字起こしテキスト
- OCR 抽出結果
- スクリーンショット候補
- FAQ 候補
- チェックリスト候補

---

## 7. MVP における `training_asset_spec` の最小要件

MVP では、フルスキーマの完全充足よりも、**正本データとして最低限成立すること**を重視します。

最低限、以下の要素が含まれていることを想定します。

### 7.1 `asset_meta`

最低限必要な情報:

- タイトル
- 目的または概要
- 対象者
- 版情報または生成日時

### 7.2 `source_evidence`

最低限必要な情報:

- transcript
- 発話時刻情報
- 必要に応じた OCR 断片

### 7.3 `instructional_core`

最低限必要な情報:

- summary
- chapters
- procedures
- steps
- cautions

### 7.4 `_metadata`

最低限必要な情報:

- schema version
- generated at
- review status

---

## 8. MVP における出力品質の考え方

MVP では完璧さを目指しません。  
ただし、以下の品質は最低限必要です。

### 8.1 手順抜けが致命的でないこと

多少の粒度差は許容しても、主要手順が大きく欠けていないことを重視します。

### 8.2 新人が追える粒度であること

経験者しか理解できない省略ではなく、初見でも追える程度の粒度を目指します。

### 8.3 注意点が落ちすぎていないこと

事故防止やミス防止に関わる情報が丸ごと消えていないことが重要です。

### 8.4 元動画へ戻れること

必要に応じて transcript や timestamp から元動画を確認できる状態を保ちます。

### 8.5 出力が読めること

HTML / PDF が最低限マニュアルとして読める構成になっていることを重視します。

---

## 9. MVP の受け入れ条件

MVP の受け入れ条件は、以下を基本とします。

### 9.1 構造面

- 単一動画から `training_asset_spec` が生成される
- JSON の主要フィールドが欠落していない
- 章・手順・ステップの最低限の階層が存在する
- HTML / PDF が生成される

### 9.2 内容面

- 動画の主要な手順が反映されている
- 要約と手順が混ざりすぎていない
- 注意点が最低限抽出されている
- 手順が極端に抽象的すぎない

### 9.3 証跡面

- transcript が保持されている
- 少なくとも一部の手順が時刻情報と対応づけられている
- 必要に応じて OCR や screenshot candidate を参照できる

### 9.4 利用面

- HTML / PDF をレビューに回せる
- 人手で修正・改善の指示を出せる
- 次回改善につながる確認ができる

---

## 10. MVP でやらないこと

MVP では、以下は非対象とします。

- 複数動画の統合マニュアル化
- 高度なブランド対応デザイン
- 承認ワークフローの実装
- 管理画面の実装
- API 提供
- 高度な検索システム連携
- 完全自動レビュー
- 全業種・全形式の動画への汎用対応
- 大規模バッチ処理の最適化

---

## 11. MVP におけるリスク

MVP では、以下のリスクがある前提で進めます。

### 11.1 要約が強すぎて手順が消える

説明の整理を優先しすぎると、実務に必要な操作が落ちる可能性があります。

### 11.2 手順粒度が粗すぎる

動画の内容を雑にまとめると、新人が再現できない資料になります。

### 11.3 注意点の抽出漏れ

手順だけ取れても、事故防止や NG 例が抜けると実務で危険です。

### 11.4 証跡との紐付け不足

構造化できても、元動画へ戻れないとレビューや改善が難しくなります。

### 11.5 PDF 見た目に引っ張られる

初期段階で見た目に注力しすぎると、本質である構造化品質の改善が遅れます。

---

## 12. MVP 実装の優先順位

MVP では、以下の順で進めることを推奨します。

### Step 1

単一動画を入力として扱えるようにする

### Step 2

transcript と最低限の OCR を取得する

### Step 3

説明 / 操作 / 注意点を最低限分類する

### Step 4

要約と手順を別処理で抽出する

### Step 5

`training_asset_spec` に正規化する

### Step 6

HTML / PDF マニュアルを出力する

### Step 7

人手レビューで改善点を確認する

---

## 13. MVP 完了の判断基準

次の状態になれば、MVP はひとまず成立とみなします。

- 対象動画 1 本から成果物一式を生成できる
- JSON が正本として扱える最低限の構造を持つ
- 要約と手順抽出が明確に分かれている
- HTML / PDF がレビュー可能な形で出力される
- 手順抜け・注意点不足・証跡接続について改善ポイントを把握できる

---

## 14. MVP 後の次段階

MVP 完了後は、以下のような拡張が候補になります。

- FAQ 候補生成の強化
- チェックリスト出力の強化
- スクリーンショット活用の改善
- 複数動画の統合
- レビュー補助の強化
- 検索やナレッジベース向け再利用
- ブランドテンプレート対応

---

## 15. まとめ

VideoAsset Manualize の MVP は、  
**1本の動画から、構造化 JSON・要約・HTML / PDF マニュアルを安定して生成できること**を目標にします。

ここで重視するのは、機能数の多さではなく、以下の基本原則が成立していることです。

- `training_asset_spec` を正本とする
- 要約と手順抽出を分離する
- 証跡を保持する
- HTML / PDF は派生生成とする
- 人手レビュー可能な品質で出力する

この MVP が成立すれば、本プロジェクトはその先の FAQ、チェックリスト、検索、複数動画統合へ段階的に拡張できる土台を持つことになります。
